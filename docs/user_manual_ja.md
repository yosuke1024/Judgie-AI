# ⚖️ Judgie-AI 利用者マニュアル (User Manual)

Judgie-AI は、Google Gemini のマルチモーダルAIを活用して、提出物（ソースコード、デモ動画、プレゼンスライド、職務経歴書など）を多角的な視点から自動評価し、実践的なフィードバックをチームに提供する汎用的なプロジェクト評価プラットフォーム（マルチテナント）です。

このマニュアルでは、システムの起動後、各ユーザーロール（**Super Admin**, **Project Admin (プロジェクト管理者)**, **Team / Participant (一般参加者・候補者)**, **Observer (オブザーバー / 観客)**）がどのようにプラットフォームを利用すべきか、ステップバイステップで説明します。

## 1. 共通の初期起動とログイン方法

### 起動方法
Judgie-AI が正常に起動されると、フロントエンド（React/Vite）のログイン画面がブラウザで開きます（デフォルトでは `http://localhost:5173` または `http://localhost:5173/login`）。

### 言語切り替え
ログイン画面およびアプリ内にある言語選択ボタン（地球儀アイコン）から、UIおよびAI生成フィードバックの表示言語をいつでも切り替えることができます。

### ✨ デモ体験モード (Demo Experience)
ログイン情報やAPIキーの設定を行わずに、すぐにシステムの機能を体験することができます。
ログイン画面の「✨ デモ体験」セクションから、以下のIDとパスコードを入力してログイン可能です。
- **一般参加者として体験**: ユーザーID `demo_team` / パスコード `demo123`
- **管理者・ホストとして体験**: ユーザーID `demo_admin` / パスコード `demo123`
- > [!NOTE]
  > デモモード（プロジェクトID: 9999）は安全な **閲覧専用（Read-only）** モードです。ファイルのアップロード、AI評価の実行、設定の変更やユーザーの追加などの書き込み操作は制限されています。あらかじめ用意された評価結果やチャット履歴をお楽しみください。

## 2. 🌍 Super Admin (全体管理者) ガイド

Super Admin は、システム全体の管理、新しいプロジェクト（テナント）の作成、およびプロジェクト管理者アカウントの発行を担当します。

### 2-1. 初回ログインとセキュリティ設定
1. ログイン画面下部にある **「🌍 Super Admin Login」** リンクをクリックし、Super Adminログイン画面に移動します。
2. 以下のデフォルト認証情報を使用してログインします：
   - **Super Admin ID:** `superadmin`
   - **パスコード:** `superadmin123`
3. ログイン後、**「🌍 Super Admin Console」** が表示されます。
4. > [!CAUTION]
   > セキュリティのため、ログイン後は **「Change Password (パスコード変更)」** セクションから必ず初期パスコードを変更してください。

### 2-2. 新規プロジェクト（テナント）の作成
1. **「Create New Project (新規プロジェクト作成)」** フォームへ進みます。
2. 以下の情報を入力します：
   - **プロジェクト名 (Project Name):** 例「Summer AI Project 2026」
   - **テナント管理者ID (Tenant Admin ID):** 例 `admin`
   - **テナント管理者パスコード (Tenant Admin Password):** プロジェクト管理者がログインに使用するパスコード
3. **「Create Project」** ボタンをクリックして作成します。データベース上に完全に分離されたテナント領域が作成されます。

### 2-3. テナントの管理と削除
1. 登録されているプロジェクト一覧から、テナントの削除や、紛失したテナント管理者のパスワード再設定などを行えます。
2. > [!WARNING]
   > プロジェクトを削除すると、そのプロジェクトに紐づくチーム/参加者情報、提出物履歴、AIの評価ログなどすべてのデータが完全に削除され、復旧できません。実行する際は十分にご注意ください。

## 3. 👑 Project Admin (プロジェクト管理者) ガイド

プロジェクト主催者・管理者は、Super Admin から共有された管理者アカウントを使用してログインし、プロジェクト環境の構築、評価テンプレート・AI審査員の設定、参加者・チームの登録、および審査結果の確認を行います。

### 3-1. 管理者としてログイン
1. 通常のログイン画面（`http://localhost:5173/login`）へアクセスします。
2. **「Select Project（プロジェクトを選択）」** で、管理対象 of プロジェクトを選択します。
3. **「Team ID / Admin ID」** に管理者ID（例: `admin`）とパスコードを入力し、**「Log In」** をクリックします。

### 3-2. プロジェクトのセットアップ手順

ログイン後、**「👑 Admin Command Center」** にアクセスできます。以下の順序でセットアップを進めてください。

#### 🚀 ステップ1: 評価テンプレートの選択 (Project Setup: Select Evaluation Template)
プロジェクトの作成後、管理者が初めてログインした際、初期化のための **「評価テンプレート」** の選択画面が表示されます（デモモードを除く）。
以下のビルトインテンプレートから用途に合わせて選択するか、独自のテンプレートを読み込ませることができます。

| テンプレート名 | 用途・説明 | 初期コンテキストモード | デフォルトQ&Aターン数 |
| :--- | :--- | :--- | :--- |
| **プロジェクト審査 (Hackathon Evaluation)** | プロトタイプの完成度、技術実装、UI/UX、プレゼンを多角的に評価する開発ハッカソン向け設定。 | 累積（イテレーション） | 1ターン (1往復のみ) |
| **スタートアップピッチ審査 (Startup Pitch Review)** | 市場性、ビジネスモデル、競合優位性、チーム実行力を審査するピッチコンテスト向け設定。 | 独立（毎回新規採点） | 3ターン |
| **採用・技術面接評価 (Hiring & Technical Interview)** | 技術スキル、システム設計能力、コミュニケーション、チーム適合性をチェックする採用面接向け設定。 | 独立（毎回新規採点） | 5ターン |
| **ソフトウェアアーキテクチャレビュー (Software Architecture Review)** | スケーラビリティ、信頼性、セキュリティ、コスト効率を厳格に評価するシステム設計監査向け設定。 | 独立（毎回新規採点） | 0ターン（Q&A無効） |
| **カスタム (Import from URL)** | 独自の評価軸やAIペルソナを定義したJSONファイルを外部から読み込む設定。 | - | - |

- **カスタムテンプレートのインポート**:
  - 独自の評価テンプレート（JSON形式）をGitHubなどの公開サーバーに配置し、その **Raw URL**（例: `https://raw.githubusercontent.com/.../template.json`）を指定して読み込ませることができます。
  - 安全のため、インポートできるドメインは許可された信頼性の高いドメイン（`github.com`, `raw.githubusercontent.com` など）に限られます。

#### ステップ2: システム設定（API接続テスト・モデル選択）
- **「System Settings (システム設定)」** ページを開きます。
- **「🤖 Gemini設定」** タブで、プロジェクト用の `Gemini APIキー` を入力し、**「APIキーを検証して保存」** ボタンをクリックします。
  - システムが自動的に接続検証を行い、そのキーで現在利用可能なモデルリストを動的に取得・保存します。
- キーの検証に成功すると、**「モデルとプランの設定」** セクションが展開されます。
  - **プラン選択:** お使いのAPIキーのプラン（無料枠 / 有料枠）を選択します。選択されたプランに合わせてアドバイスが表示されます。
  - **モデル選択:** APIから取得された利用可能モデル一覧（例: `gemini-3.5-flash` や `gemini-3.1-pro`）から、使用するモデルを選択して保存します。
  - > [!IMPORTANT]
    > 本番運用時はリクエスト制限（Rate Limits）のクォータ回避のために **有料枠（Paid Tier）** のキーを使用することを強く推奨します。無料枠の場合は `gemini-3.5-flash` または `gemini-3.1-flash-lite` を選んでください。

#### ステップ3: 評価軸の設定（Evaluation Criteria）
- **「⚖️ Evaluation Criteria」** タブへ進みます。
- AI審査員がスコア（1〜5点）を算出する際の基準となる評価軸を登録・編集します。
- 各評価軸には **「重み（Weight %）」** を設定でき、総合スコア（100点満点）の算出に反映されます。

#### ステップ4: 審査員ペルソナの設定（Judges (Personas)）
- **「🧑‍🏫 Judges (Personas)」** タブへ進みます。
- 評価を担当するAI専門家のペルソナ（例：UXデザイナー、VC、インフラエンジニアなど）を作成・編集できます。
- 同時に有効化（Active）できるAI審査員は **最大5名** までです。
- カスタムアバター画像（PNG/JPG, 最大500KB）のアップロードも可能です。

#### ステップ5: チーム・参加者の登録（Team Management）
- **「🏢 Team Management」** タブへ進みます。一般参加者（team）または閲覧専用ユーザー（observer）のアカウントを登録します。
- **個別追加**: 画面上のフォームから `ユーザーID`、`パスコード`、および **`ロール (Role)`** を選択して追加します。
- **CSV一括登録**: CSVファイルをアップロードして一括インポートが可能です。
  - フォーマット: `team_id, passcode, role`（3列目に `observer` または `team` を指定可能。省略時は `team` になります）。
- 登録完了後、各ユーザーにログイン情報を配布してください。

#### ステップ6: AIレスポンスの設定（AI Response Settings）
- **「🤖 AIレスポンス設定」** タブへ進みます。
- AIのフィードバック、最優先アクション、反論回答、AdminチャットにおいてAIが出力する言語を、最大5つまで設定・保存できます（例: English, Japanese）。
- > [!WARNING]
  > 言語設定を増やすと、AIがそれぞれの言語表現で評価テキストを翻訳・生成するため、解析完了までの時間（遅延）が長くなります。最適なパフォーマンスのために、必要な言語のみ（通常は1〜2つ）を設定することを推奨します。

#### ステップ7: プロジェクト詳細・挙動設定（Project Settings）
- **「⚙️ Project Settings」** タブへ進みます。このプロジェクト専用の挙動と対話制限を微調整します。
- **🔄 再評価（イテレーション）時のコンテキスト**:
  - **累積 (Cumulative)**: 成果物が再アップロードされた際、前回の評価内容や改善ポイントをコンテキストに含め、改善度を評価に反映させます（継続的なイテレーション型プロジェクト推奨）。
  - **独立 (Independent)**: 前回の評価は引き継がず、毎回純粋に提出物のみでフラットに新規採点します（採用や単発の設計レビュー推奨）。
- **🙋 Q&A（異議申し立て）の最大ターン数**:
  - 一般参加者が、評価結果に対して最大何回AI審査員へ質問や反論を送信できるかを設定します（Q&A無効、1〜5往復、無制限）。

### 3-3. プロジェクト進行中の運用

#### 進行状況とリアルタイムスコアボードの確認
- **「📊 Live Scoreboard」** タブから、各チームの最新の総合スコア、タイブレーカー用スコア（Impact）、AI相談（Consultation）の利用回数を確認できます。

#### 提出物とAI個別チャット機能
- **「💬 Submissions & AI Chat」** タブへ進みます。
- 任意のチームと、過去の提出履歴を選択します。
- チームが提出した実際のソースコードや資料に基づき、AI審査員パネルへ直接質問ができます。
  - *質問例: 「このチームのシステムアーキテクチャのボトルネックはどこですか？」「セキュリティの脆弱性は見つかりましたか？」*
  - この機能により、運営者は審査の裏付けや技術的難易度の把握をAIの支援を受けて行うことができます。

## 4. 🧑‍💻 Team (一般参加者・候補者) ガイド

参加者は、運営者から配布されたアカウントでログインし、開発中のプロダクトの中間評価（AI Coaching）を受けたり、最終提出を行ったりします。

### 4-1. チームとしてログイン
1. ログイン画面（`http://localhost:5173` または `http://localhost:5173/login`）へアクセスします。
2. **「Select Project」** で参加するプロジェクトを選択します。
3. 運営者から配布された **「Team ID」** と **「Passcode」** を入力し、ログインします。

### 4-2. プロフィールおよびパスワードの管理
- **「Team Profile」** フォームから、プロダクト名（Product Name）、チーム名（Team Name）、一言キャッチコピー（One-liner Pitch）を設定・更新できます。入力後は「Save」ボタンをクリックして保存します。
- **「🔐 Change Password」** から、自身でパスコードを変更可能です。

### 4-3. 成果物のアップロードとAI相談（AI Consultation）

本提出前に、AI審査員から中間フィードバック（AI Consultation）を受けることができます。
- AI相談を行える残り回数は、画面の「Consultations Left」に表示されます（システム環境のデフォルト設定では **最大3回** です）。

#### 📁 対応している提出形式とAIによる処理
提出物は以下の形式に対応しており、プロジェクトの性質に合わせて **任意の組み合わせで提出可能** です（ソースコードZIPが必須ではないため、PDFスライドやデモ動画のみでの審査も可能です）。

1. **ソースコード等の圧縮ファイル (ZIP):**
   - 主にソースコードやテキストドキュメントの提出用です。
   - アップロードされると、システムが自動的にテキストを抽出し、AI審査員に解析用テキストコンテキストとして渡します。
   - > [!TIP]
     > `node_modules` や `.git`、`venv` などの巨大な依存・管理用ディレクトリは、アップロード制限（合計200MB）やAIのトークン制限を超えないよう、必ずZIP化の対象から除外してください。
     > Mac/Linuxでの除外ZIP作成コマンド例: `zip -r submission.zip . -x "node_modules/*" -x ".git/*" -x "venv/*" -x ".next/*"`
2. **プレゼンテーション資料 (PDF):**
   - 企画書、デザインモック、プレゼンスライド、あるいは職務経歴書用です。
   - Geminiのドキュメント理解により、レイアウトや文字、図表がそのまま視覚的に解析・評価されます。
3. **デモ動画 (MP4 / MOV):**
   - プロダクトの実際の動作品やプレゼン発表の録画用です。
   - Geminiのマルチモーダル動画理解により、UIの操作感やデザイン、実際の機能動作が直接評価されます。

#### 提出と相談の手順
1. **ファイルのアップロード:**
   - 画面の「Submit Your Work」エリアに、対象のファイルをドラッグ＆ドロップまたはファイルブラウザから配置します（複数ファイルの同時アップロードが可能です）。
2. **AIコーチングを受ける (Submit for AI Consultation):**
   - **「Submit for AI Consultation」** ボタンをクリックします。AI審査員パネル（最大5名）が提出物を解析し、数分でフィードバックを生成します。
   - 相談回数は1回消費されます。

### 4-4. フィードバックとスコアの確認

解析が完了すると、**「Evaluation Results」** パネルに結果が表示されます。

- **OVERALL IMPACT SCORE:** 設定された評価軸と重みに基づく、100点満点換算の現在の評価。
- **Evaluation Balance (Radar):** 各評価軸ごとの詳細スコア（5点満点）を視覚化するレーダーチャート。
- **Score Trend (Max 100):** 相談1回目から現在までのスコアの推移（履歴）を視覚化する折れ線グラフ（過去の相談履歴は右上の「History」ドロップダウンから切り替え可能です）。
- **Criteria Breakdown (左側パネル):** 各評価軸の5点満点スコアおよびウェイトに対する貢献ポイントを表示。
- **AI Summary:** 提出されたプロダクトの概要と、AIによる総合的なサマリー。
- **AI Judges Feedback:** 有効化されている各AI審査員の個別講評（それぞれのペルソナ特有の視点や口調で出力されます）。
  - *※ 管理者が複数のAIレスポンス言語を設定している場合、ダッシュボードに言語別のタブが表示され、クリックして表示を切り替えることができます。*

### 4-5. 異議申し立て・質問機能（"Objection!"）
AIのスコアやフィードバックに疑問がある、またはより深く質問したい場合、管理者が設定した最大Q&A対話ターン数（デフォルトは1往復）を上限として、審査員に反論・質問を送ることができます。

1. フィードバック画面下部の **「🙋 Objection! / Q&A」** セクションへ進みます。
2. フォームに質問や反論を記述し、**「Send Message ✊」** ボタンをクリックします。
3. AI審査員同士がディベートを行い、あなたへの合同回答を生成します。ディベートの結果、フィードバックが更新される場合があります。

### 4-6. 最終成果物としての提出 (Final Submission)
開発や課題が完了し、最終評価を受ける準備ができたら、成果物ファイルをアップロードした状態で **「Final Submission」** チェックボックスにチェックを入れます。
ボタンが「Submit Final Submission」に切り替わりますので、クリックして最終提出を実行します。
- 最終提出を行うと、それ以降の成果物の再アップロードやAI相談は行えなくなります。
- 最終提出のスコアが、運営者のスコアボードに「最終スコア」として固定されます。

## 5. 👁️ Observer (オブザーバー / 観客) ガイド

オブザーバー（観客）は、プロジェクト管理者から発行されたアカウントを使用してログインし、プロジェクト内の各チームの進行状況やスコアボードを安全に監視・閲覧します。

### 5-1. オブザーバーとしてログイン
1. 通常のログイン画面（`http://localhost:5173/login`）へアクセスします。
2. **「Select Project」** で対象のプロジェクトを選択します。
3. 管理者から配布された **「Observer ID」** と **「Passcode」** を入力し、ログインします。

### 5-2. 閲覧可能エリアと機能制限（閲覧専用モード）
ログイン後、一般参加者（Team）と類似したUIにアクセスできますが、画面上部には **「Observer Mode: Read-Only View」** と表示され、一切の書き込み・変更操作が制限されます。

- **閲覧可能な情報**:
  - **チームダッシュボードの閲覧**: 左側のサイドバーナビゲーションメニューにある各チーム名リンク（「TEAM DASHBOARDS」エリア）から対象のチームを切り替え、各チームが提出した過去の成果物の評価結果、AIフィードバック、ディベート履歴（Objection!）などを詳細に閲覧できます。
  - **リーダーボードの閲覧**: プロジェクトに登録されている全チームのリアルタイムなスコアランキングや、インパクトスコア、AI相談の利用回数を確認できます。Category Leadersタブで部門別のリーダーボードランキングも確認可能です。
- **制限される操作 (書き込み不可)**:
  - プロフィールの編集（Team ProfileのSave）
  - パスワード変更（Change Password）
  - 成果物ファイルのアップロード
  - AI相談の実行（Submit for AI Consultation）

## 6. ❓ FAQ & トラブルシューティング

#### Q. ZIPファイルのアップロードでエラーになります。
- **A.** 合計ファイルサイズが200MBを超えていないか確認してください。特にソースコードのZIP化の際、`node_modules` や `.git` などのビルド・管理用フォルダが含まれたままになっていると容量制限に引っかかります。これらを除外して再度ZIPを作成してください。

#### Q. 提出したソースコードの一部がAIに読み込まれていない、またはハルシネーション（嘘）が起きているように見えます。
- **A.** Judgie-AIは、AIの入力コンテキスト限界やAPIの制限を回避するため、ZIP内から抽出するコードテキストの総文字数を **最大800,000文字（約20万トークン）** に制限しています。これを超える大容量のソースコードが提出された場合、システムは自動的にテキストの末尾を切り捨て、`[SYSTEM WARNING: Codebase too large...]` という警告を挿入します。不要なライブラリやデータファイル、アセットなどはZIP化の対象から除外してください。

#### Q. AI相談を実行した際、応答に時間がかかります。
- **A.** デモ動画の解析や大容量コードの読み込みを行っている場合、Gemini API側の処理に1〜3分程度かかる場合があります。画面上の「Processing...」表示が消えるまでブラウザを閉じずにそのままお待ちください。

#### Q. 初期パスコード `superadmin123` で Super Admin にログインできません。
- **A.** 既に別の全体管理者がパスコードを変更しているか、初期設定がリセットされている可能性があります。サーバーの再起動またはデータベースの初期化が必要な場合があります。

#### Q. AIの回答がハルシネーション（嘘）を起こしているように見えます。
- **A.** 提出されたZIPファイルが破損している、またはコードの依存関係が極めて特殊な場合、AIが正しく内容をパースできず推測で回答する場合があります。ZIPファイルの中に主要なソースコード（`app.js`, `main.py` など）がルートまたは分かりやすい位置に含まれているか確認してください。
