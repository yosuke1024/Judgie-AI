# ⚖️ Judgie-AI 利用者マニュアル (User Manual)

Judgie-AI は、Google Gemini のマルチモーダルAIを活用して、ハッカソンの提出物（ソースコード、デモ動画、プレゼンスライドなど）を多角的な視点から自動評価し、実践的なフィードバックをチームに提供するハッカソン評価プラットフォームです。

このマニュアルでは、システムの起動後、各ユーザーロール（**Super Admin**, **Hackathon Admin**, **Team / Participant**）がどのようにプラットフォームを利用すべきか、ステップバイステップで説明します。

## 1. 共通の初期起動とログイン方法

### 起動方法
Judgie-AI が正常にデプロイまたはローカルで起動されると、ブラウザで Streamlit のログイン画面が開きます（デフォルトでは `http://localhost:8501`）。

### 言語切り替え
画面左側のサイドバー上部にある **「Language / 言語」** ラジオボタンから、UIおよびAI生成フィードバックの表示言語（**English / 日本語**）をいつでも切り替えることができます。

## 2. 🌍 Super Admin (全体管理者) ガイド

Super Admin は、システム全体の管理、新しいハッカソン（テナント）の作成、およびハッカソン管理者アカウントの発行を担当します。

### 2-1. 初回ログインとセキュリティ設定
1. ログイン画面のURLに `?admin=true` パラメータを付与してアクセスします（例: `http://localhost:8501/?admin=true`）。
2. 以下のデフォルト認証情報を使用してログインします：
   - **Super Admin ID:** `superadmin`
   - **パスコード:** `superadmin123`
3. ログイン後、**「🌍 Super Admin Console」** が表示されます。
4. > [!CAUTION]
   > セキュリティのため、ログイン後は **「Change Password (パスコード変更)」** セクションから必ず初期パスコードを変更してください。

### 2-2. 新規ハッカソン（テナント）の作成
1. **「Create New Hackathon (新規ハッカソン作成)」** フォームへ進みます。
2. 以下の情報を入力します：
   - **ハッカソン名 (Hackathon Name):** 例「Summer AI Hackathon 2026」
   - **テナント管理者ID (Admin User ID):** 例 `admin`
   - **管理者パスコード (Admin Passcode):** ハッカソン管理者がログインに使用するパスコード
3. **「Create Hackathon」** ボタンをクリックして作成します。データベース上に完全に分離されたテナント領域が作成されます。

### 2-3. テナントの管理と削除
1. 登録されているハッカソン一覧から、ハッカソンの削除や、紛失したテナント管理者のパスワード再設定などを行えます。
2. > [!WARNING]
   > ハッカソンを削除すると、そのハッカソンに紐づくチーム情報、提出物履歴、AIの評価ログなどすべてのデータが完全に削除され、復旧できません。実行する際は十分にご注意ください。

## 3. 👑 Hackathon Admin (ハッカソン運営者) ガイド

ハッカソン運営者は、Super Admin から共有された管理者アカウントを使用してログインし、ハッカソン環境の構築、評価基準・AI審査員の設定、参加チームの登録、および審査結果の確認を行います。

### 3-1. 管理者としてログイン
1. 通常のログイン画面（`http://localhost:8501`）へアクセスします。
2. **「Select Hackathon（ハッカソンを選択）」** で、自身のハッカソンを選択します。
3. **「Team ID / Admin ID」** に管理者ID（例: `admin`）とパスコードを入力し、**「ログイン」** をクリックします。

### 3-2. ハッカソンのセットアップ手順

ログイン後、**「👑 Admin Command Center」** にアクセスできます。以下の順序でセットアップを進めてください。

#### ステップ1: システム設定（API接続テスト・モデル選択）
- **「System Settings (システム設定)」** ページを開きます。
- **「🤖 Gemini設定」** タブで、ハッカソン用の `Gemini APIキー` を入力し、**「APIキーを検証して保存」** ボタンをクリックします。
  - システムが自動的に接続検証を行い、そのキーで現在利用可能なモデルリストを動的に取得・保存します。キーが無効な場合はエラーになります。
- キーの検証に成功すると、**「モデルとプランの設定」** セクションが展開されます。
  - **プラン選択:** お使いのAPIキーのプラン（無料枠 / 有料枠）を選択します。選択されたプランに合わせてアドバイスが表示されます。
  - **モデル選択:** APIから取得された利用可能モデル一覧（例: `gemini-3.5-flash` や `gemini-3.1-pro`）から、使用するモデルを選択して保存します。
  - > [!IMPORTANT]
    > 本番運用時はリクエスト制限（Rate Limits）のクォータ回避のために **有料枠（Paid Tier）** のキーを使用することを強く推奨します。無料枠の場合は `gemini-3.5-flash` または `gemini-3.1-flash-lite` を選んでください。

#### ステップ2: 評価軸の設定（Evaluation Criteria）
- **「⚖️ Evaluation Criteria」** タブへ進みます。
- AI審査員がスコア（1〜5点）を算出する際の基準となる評価軸を登録します（例：`技術的な新規性`, `ビジネスモデルの実現性`, `UI/UXの完成度` など）。
- 各評価軸には **「重み（Weight %）」** を設定でき、総合スコア（100点満点）の算出に反映されます。
- **「Detailed Description (詳細説明)」** には、AIがどのような視点で評価すべきか、プロンプトとなる指示を詳細に入力します。

#### ステップ3: 審査員ペルソナの設定（Judges (Personas)）
- **「🧑‍🏫 Judges (Personas)」** タブへ進みます。
- 評価を担当するAI専門家のペルソナ（例：UXデザイナー、VC、インフラエンジニアなど）を作成・編集できます。
- 同時に有効化（Active）できるAI審査員は **最大5名** までです。チェックボックスで有効/無効を切り替えます。
- **プロンプト定義** には、その人物の役職、口調、重要視する観点などを詳細に記述してください。カスタムアバター画像（PNG/JPG, 最大500KB）のアップロードも可能です。

#### ステップ4: チーム登録（Team Management）
- **「🏢 Team Management」** タブへ進みます。
- **CSV一括登録:** `team_id, passcode` の形式で作成したCSVファイルをアップロードして一括インポートが可能です。
- **個別追加:** 画面上のフォームから `チームID` と `パスコード` を入力して手動で追加します。
- 登録完了後、各チームにログイン情報（ハッカソン名、チームID、パスコード）を配布してください。

### 3-3. ハッカソン開催中の運用

#### 進行状況とリアルタイムスコアボードの確認
- **「📊 Live Scoreboard」** タブから、各チームの最新の総合スコア、タイブレーカー用スコア（Impact）、AI相談（Consultation）の利用回数を確認できます。

#### 提出物とAI個別チャット機能
- **「💬 Submissions & AI Chat」** タブへ進みます。
- 任意のチームと、過去の提出履歴を選択します。
- チームが提出した実際のソースコードや資料に基づき、AI審査員パネルへ直接質問ができます。
  - *質問例: 「このチームのシステムアーキテクチャのボトルネックはどこですか？」「バックエンドで使用されているフレームワークは何ですか？」「セキュリティの脆弱性は見つかりましたか？」*
  - この機能により、運営者は審査の裏付けや技術的難易度の把握をAIの支援を受けて行うことができます。

## 4. 🧑‍💻 Team (ハッカソン参加チーム) ガイド

参加者は、運営者から配布されたアカウントでログインし、開発中のプロダクトの中間評価（AI Consultation）を受けたり、最終提出を行ったりします。

### 4-1. チームとしてログイン
1. ログイン画面（`http://localhost:8501`）へアクセスします。
2. **「Select Hackathon」** で参加するハッカソンを選択します。
3. 運営者から配布された **「Team ID」** と **「Passcode」** を入力し、ログインします。

### 4-2. プロフィールおよびパスワードの管理
- **「📤 Submission」** 画面の **「⚙️ Edit Profile」** ポップオーバーから、プロダクト名、チーム名、一言キャッチコピー（One-liner）を設定・更新できます。
- **「🔐 Change Password」** から、自身でパスコードを変更可能です。

### 4-3. 成果物のアップロードとAI相談（AI Consultation）

Judgie-AI では、本提出前に **最大3回** までAI審査員から中間フィードバックを受けることができます。

#### 📁 対応している提出形式とAIによる処理
提出物は以下の形式に対応しており、ハッカソンの性質（開発ハッカソン、アイデアソン、ノーコード等）に合わせて **任意の組み合わせで提出可能** です（ソースコードZIPが必須ではないため、PDFスライドやデモ動画のみでの審査も可能です）。

1. **ソースコード等の圧縮ファイル (ZIP):**
   - 主にソースコードやテキストドキュメントの提出用です。
   - アップロードされると、システムが自動的にテキストを抽出し、AI審査員に解析用テキストコンテキストとして渡します。
   - > [!TIP]
     > `node_modules` や `.git`、`venv` などの巨大な依存・管理用ディレクトリは、アップロード制限（合計200MB）やAIのトークン制限を超えないよう、必ずZIP化の対象から除外してください。
     > Mac/Linuxでの除外ZIP作成コマンド例: `zip -r submission.zip . -x "node_modules/*" -x ".git/*" -x "venv/*"`
2. **プレゼンテーション資料 (PDF):**
   - 企画書、デザインモック、プレゼンスライド用です。
   - Geminiのドキュメント理解により、レイアウトや文字、図表がそのまま視覚的に解析・評価されます。
3. **デモ動画 (MP4 / MOV):**
   - プロダクトの実際の動作品やプレゼン発表の録画用です。
   - Geminiのマルチモーダル動画理解により、UIの操作感やデザイン、実際の機能動作が直接評価されます。

#### 提出と相談の手順
1. **ファイルのアップロード:**
   - 画面の「成果物ファイル (ZIP, MP4, MOV, PDF)」エリアに、対象のファイルをドラッグ＆ドロップまたはファイルブラウザから配置します（複数ファイルの同時アップロードが可能です）。
2. **AIコーチングを受ける (Get AI Coaching):**
   - **「Get AI Coaching」** ボタンをクリックします。AI審査員パネル（最大5名）が提出物を解析し、数分でフィードバックを生成します。
   - 相談回数は1回消費されます。

### 4-4. フィードバックとスコアの確認

解析が完了すると、**「💬 AI Feedback Dashboard」** に結果が表示されます。

- **総合スコア (Total Score):** 設定された評価軸と重みに基づく、100点満点換算の現在の評価。
- **スコア内訳:** 各評価軸ごとの詳細スコア（5点満点）と、前回の相談時からのスコア変動（デルタ値）。
- **推移グラフ:** 相談1回目から現在までのスコアの伸びを視覚化するチャート。
- **フィードバックタブ:**
  - **最優先アクション (Next Steps):** 次に何を改善すべきか、AIが導き出した具体的なアクションリスト。
  - **プロダクト理解 (AI Product Understanding):** AIが認識したプロダクトの概要。
  - **審査員フィードバック (Judges Feedback):** 有効化されている各AI審査員の個別講評（それぞれのペルソナ特有の視点や口調で出力されます）。

### 4-5. 異議申し立て・質問機能（"Objection!"）
AIのスコアやフィードバックに疑問がある、またはより深く質問したい場合、各相談結果につき **1回限り** 審査員に反論・質問を送ることができます。

1. フィードバック画面下部の **「🙋 Objection! / Q&A」** セクションへ進みます。
2. フォームに質問や反論を記述し、**「Objection! ✊」** ボタンをクリックします。
3. AI審査員同士がディベートを行い、あなたへの合同回答を生成します。ディベートの結果、フィードバックが更新される場合があります。

### 4-6. 最終成果物としての提出 (Final Submission)
開発が完了し、最終評価を受ける準備ができたら、成果物ファイルをアップロードした状態で **「Submit Final Pitch (最終成果物として提出)」** をクリックします。
- 最終提出を行うと、それ以降の成果物の再アップロードやAI相談は行えなくなります。
- 最終提出のスコアが、運営者のスコアボードに「最終スコア」として固定されます。

## 5. ❓ FAQ & トラブルシューティング

#### Q. ZIPファイルのアップロードでエラーになります。
- **A.** 合計ファイルサイズが200MBを超えていないか確認してください。特にソースコードのZIP化の際、`node_modules` や `.git` などのビルド・管理用フォルダが含まれたままになっていると容量制限に引っかかります。これらを除外して再度ZIPを作成してください。

#### Q. AI相談を実行した際、応答に時間がかかります。
- **A.** デモ動画の解析や大容量コードの読み込みを行っている場合、Gemini API側の処理に1〜3分程度かかる場合があります。画面上の「Processing...」表示が消えるまでブラウザを閉じずにそのままお待ちください。

#### Q. 初期パスコード `superadmin123` で Super Admin にログインできません。
- **A.** 既に別の全体管理者がパスコードを変更しているか、初期設定がリセットされている可能性があります。サーバーの再起動またはデータベースの初期化が必要な場合があります。

#### Q. AIの回答がハルシネーション（嘘）を起こしているように見えます。
- **A.** 提出されたZIPファイルが破損している、またはコードの依存関係が極めて特殊な場合、AIが正しく内容をパースできず推測で回答する場合があります。ZIPファイルの中に主要なソースコード（`app.js`, `main.py` など）がルートまたは分かりやすい位置に含まれているか確認してください。
