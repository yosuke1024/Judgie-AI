# ⚖️ Judgie-AI 利用者マニュアル (User Manual)

Judgie-AI は、Google Gemini などのマルチモーダルAIを活用して、提出物（ソースコード、デモ動画、プレゼンスライド、職務経歴書など）を多角的な視点から自動評価し、実践的なフィードバックをチームに提供する汎用的なプロジェクト評価プラットフォーム（マルチテナント）です。

このマニュアルでは、システムの起動後、各ユーザーロール（**Super Admin**, **Project Admin (プロジェクト管理者)**, **Team / Participant (一般参加者・候補者)**, **Observer (オブザーバー / 観客)**）がどのようにプラットフォームを利用すべきか、ステップバイステップで説明します。

## 1. 共通の初期起動とログイン方法

### 起動方法
Judgie-AI が正常に起動されると、フロントエンド（React/Vite）のログイン画面がブラウザで開きます（デフォルトでは `http://localhost:5173` または `http://localhost:5173/login`）。

### 言語切り替え
ログイン画面およびアプリ内にある言語選択ボタン（地球儀アイコン）から、UIおよびAI生成フィードバックの表示言語をいつでも切り替えることができます。

### ログイン認証の種類
システム設定（環境変数）に応じて、ログイン方法が自動的に切り替わります。
1. **パスコード認証 (デフォルト)**
   - 運営者から配布された「チームID / 管理者ID」と「パスコード」を入力してログインします。
2. **OIDC (シングルサインオン) 認証**
   - 環境変数 `OIDC_ENABLED=true` が設定されている場合、セキュリティとUX向上のため、パスコードの入力欄は非表示となり、**「Sign in with SSO (シングルサインオンでログイン)」** ボタンが表示されます。
   - ボタンをクリックして Google OAuth などのプロバイダで認証を行うと、検証されたメールアドレスに基づいて自動的に該当プロジェクト・ロールでログインが完了します。
   - OIDC有効時は、アプリケーション内の「パスコード変更」などのパスワード関連機能はすべて安全に非表示化されます。

### ✨ デモ体験モード (Demo Experience)
ログイン情報やAPIキーの設定を行わずに、すぐにシステムの機能を体験することができます。
ログイン画面の「✨ デモ体験」セクションから、以下のIDとパスコードを入力してログイン可能です（OIDC環境下でもデモ体験はご利用いただけます）。
- **一般参加者として体験**: ユーザーID `demo_team` / パスコード `demo123`
- **管理者・ホストとして体験**: ユーザーID `demo_admin` / パスコード `demo123`
- > [!NOTE]
  > デモモード（プロジェクトID: 9999）は安全な **閲覧専用（Read-only）** モードです。ファイルのアップロード、AI評価の実行、設定の変更やユーザーの追加・削除などの書き込み操作は制限されています。

## 2. 🌍 Super Admin (全体管理者) ガイド

Super Admin は、システム全体の管理、新しいプロジェクト（テナント）の作成、およびプロジェクト管理者アカウントの発行を担当します。

### 2-1. 初回ログインとセキュリティ設定
1. ログイン画面下部にある **「🌍 Super Admin Login」** リンクをクリックし、Super Adminログイン画面に移動します。
2. 以下のデフォルト認証情報を使用してログインします：
   - **Super Admin ID:** `superadmin`
   - **パスコード:** `superadmin123`
3. ログイン後、**「🌍 Super Admin Console」** が表示されます。
4. > [!CAUTION]
   > セキュリティのため、ログイン後は **「Change Password (パスコード変更)」** セクションから必ず初期パスコードを変更してください（※OIDC有効時はパスコード変更画面は非表示になります）。

### 2-2. 新規プロジェクト（テナント）の作成
1. **「Create New Project (新規プロジェクト作成)」** フォームへ進みます。
2. 以下の情報を入力します：
   - **プロジェクト名 (Project Name):** 例「Summer AI Project 2026」
   - **テナント管理者ID (Tenant Admin ID):** 例 `admin`
   - **テナント管理者パスコード / メールアドレス:**
     - **通常時**: プロジェクト管理者がログインに使用するパスコードを入力します。
     - **OIDC (SSO) 有効時**: パスコード入力欄の代わりに「管理者メールアドレス (Admin Email)」を入力します。
3. **「Create Project」** ボタンをクリックして作成します。データベース上に完全に分離されたテナント領域が作成されます。

### 2-3. テナントの管理と削除
1. 登録されているプロジェクト一覧から、テナントの削除や、紛失したテナント管理者のパスワード再設定（通常時）などを行えます。
2. > [!WARNING]
   > プロジェクトを削除すると、そのプロジェクトに紐づくチーム/参加者情報、提出物履歴、AIの評価ログなどすべてのデータが完全に削除され、復旧できません。実行する際は十分にご注意ください。

## 3. 👑 Project Admin (プロジェクト管理者) ガイド

プロジェクト主催者・管理者は、Super Admin から共有された管理者アカウントを使用してログインし、プロジェクト環境の構築、評価テンプレート・AI審査員の設定、参加者・チームの登録、および審査結果の確認を行います。

### 3-1. 管理者としてログイン
1. 通常のログイン画面（`http://localhost:5173/login`）へアクセスします。
2. **「Select Project（プロジェクトを選択）」** で、管理対象のプロジェクトを選択します。
3. アカウントの種類に応じたログイン手順を踏みます：
   - **通常時**: **「Team ID / Admin ID」** に管理者ID（例: `admin`）とパスコードを入力し、**「Log In」** をクリックします。
   - **OIDC有効時**: **「Sign in with SSO」** ボタンをクリックし、ログインを完了させます。

### 3-2. プロジェクトのセットアップ手順

ログイン後、**「👑 Admin Command Center」** にアクセスできます。以下の順序でセットアップを進めてください。

#### 🚀 ステップ1: 評価テンプレートの選択 (Project Setup)
プロジェクトの作成後、管理者が初めてログインした際、初期化のための **「評価テンプレート」** の選択画面が表示されます（デモモードを除く）。
以下のビルトインテンプレートから用途に合わせて選択するか、独自のテンプレートを読み込ませることができます。

| テンプレート名 | 用途・説明 | 初期コンテキストモード | デフォルトQ&Aターン数 |
| :--- | :--- | :--- | :--- |
| **プロジェクト審査 (Hackathon Evaluation)** | プロトタイプの完成度、技術実装、UI/UX、プレゼンを多角的に評価する開発ハッカソン向け設定。 | 累積（イテレーション） | 1ターン (1往復のみ) |
| **スタートアップピッチ審査 (Startup Pitch Review)** | 市場性、ビジネスモデル、競合優位性、チーム実行力を審査するピッチコンテスト向け設定。 | 独立（毎回新規採点） | 3ターン |
| **採用・技術面接評価 (Hiring & Technical Interview)** | 技術スキル、システム設計能力、コミュニケーション、チーム適合性をチェックする採用面接向け設定。 | 独立（毎回新規採点） | 5ターン |
| **ソフトウェアアーキテクチャレビュー (Software Architecture Review)** | スケーラビリティ、信頼性、セキュリティ、コスト効率を厳格に評価するシステム設計監査向け設定。 | 独立（毎回新規採点） | 0ターン（Q&A無効） |
| **カスタム (Import from URL)** | 独自の評価軸やAIペルソナを定義したJSONファイルを外部から読み込む設定。 | - | - |

- **カスタムテンプレートのインポート**:
  - 独自の評価テンプレート（JSON形式）をGitHubなどの公開サーバーに配置し、その **Raw URL**（例: `https://raw.githubusercontent.com/.../template.json`）を指定して読み込ませることができます。
  - 安全のため、インポートできるドメインは許可された信頼性の高いドメイン（`github.com`, `raw.githubusercontent.com` など）に限られます。

#### ステップ2: システム設定（API接続テスト・モデル選択・プロバイダ）
- **「System Settings (システム設定)」** ページを開きます。
- **「🤖 Gemini設定」**（または選択されているLLMプロバイダ設定）タブで、プロジェクト用の APIキーを入力し、保存・検証します。
  - **複数LLMプロバイダの対応**: Judgie-AI は Google Gemini だけでなく、環境変数設定により OpenAI や Anthropic などの複数 LLM プロバイダでの評価にも対応しています。
  - > [!IMPORTANT]
    > Gemini 以外のプロバイダ（OpenAI, Anthropic）はマルチモーダル動画解析をサポートしていません。これらのプロバイダが選択されている場合、参加者のダッシュボードで動画ファイル（MP4/MOV）の提出が自動的に制限・ブロックされます。

#### ステップ3: 評価軸の設定（Evaluation Criteria）
- **「⚖️ 評価基準」** タブへ進みます。
- AI審査員がスコア（1〜5点）を算出する際の基準となる評価項目を登録・編集します。各評価軸には **「重み（Weight %）」** を設定でき、総合スコア（100点満点）の算出に反映されます。
- **Active / Inactive トグル**: 各評価基準のトグルスイッチを切り替えることで、削除することなく一時的に無効化（Inactive）できます。無効化された評価項目は、AI評価から自動的に除外され、全体のスコアボードの計算および表示からも除外されます。

#### ステップ4: 審査員ペルソナの設定（AI審査員）
- **「🧑‍⚖️ AI審査員」** タブへ進みます。
- 評価を担当するAI専門家のペルソナ（例：UXデザイナー、VC、インフラSREなど）を作成・編集できます。同時に有効化（Active）できるAI審査員は **最大5名** までです。
- **アバター画像の設定**: 絵文字の代わりに、カスタムのアバター画像（PNG/JPG, 最大500KB）をアップロードできます。アップロードされたアバター画像は、チームのダッシュボードや対話スレッドで綺麗な丸型画像として表示されます。
- **Active / Inactive トグル**: トグルスイッチで一時的に無効化（Inactive）できます。無効化された審査員は、AI評価およびその後のQ&A対話スレッドから一時的に除外されます。

#### ステップ5: チーム・参加者の登録（チーム管理）
- **「🏢 チーム管理」** タブへ進みます。
- **個別追加**:
  - **通常時**: `ユーザーID`、`パスコード`、および **`ロール (Role)`** を入力して登録します。
  - **OIDC (SSO) 有効時**: パスコードの代わりに「登録ユーザーのメールアドレス」を入力して登録します（パスコードはランダムで自動生成されます）。
- **CSV一括登録**: CSVファイルをアップロードして一括インポートが可能です。
  - **通常時**: `team_id, passcode, role` のフォーマット
  - **OIDC (SSO) 有効時**: `team_id, role, email` のフォーマット（パスコード不要）
- **Active / Inactive トグル**: チーム一覧の「Active」列のトグルで切り替えます。一時的に無効化（Inactive）されたチームはログインが完全に遮断され、スコアボードや提出物リストからも自動的に除外されます。
- **チーム・ユーザーの削除**: 誤登録や参加辞退したチームを、下部の「ユーザー・チームの削除」セクションから安全に完全削除（CASCADE）できます。該当する提出データやチャット履歴もすべて削除されます（※管理者の削除は禁止されています）。

#### ステップ6: AIレスポンスの設定（AI言語設定）
- **「AI言語設定」** タブへ進みます。AIが評価フィードバックやチャットで出力する言語を最大5つまで設定・保存できます。複数の言語を登録すると、ユーザーダッシュボードでタブを切り替えて各言語での評価が確認できます。

#### ステップ7: プロジェクト詳細・挙動設定（プロジェクト設定）
- **「プロジェクト設定」** タブへ進みます。イテレーション時のコンテキスト（累積か独立か）や、Q&Aの最大ターン数などの挙動を微調整します。

---

### 3-3. プロジェクト進行中の運用

#### ライブスコアボードの確認
- **「ライブスコアボード」** タブから、各チームの最新の総合スコア、AI相談の利用回数を確認できます。

#### 提出物とAI個別チャット・提出履歴の削除
- **「ディープダイブ (Submissions & AI Chat)」** タブへ進みます。
- 任意のチームと提出履歴（Consultation）を選択し、チームが提出したソースコードや動画に基づいて、AI審査員パネルへ直接質問ができます。
- **提出履歴（コンサルテーション）の個別削除**: チームが誤ったファイルを提出した際やテスト実行時、選択した提出履歴の直後にある「この提出履歴を削除」セクションから、該当する評価結果とチャット履歴を完全に削除できます。これにより、**チームの「残り相談回数」も自動的に1回復**します。

## 4. 🧑‍💻 Team (一般参加者・候補者) ガイド

参加者は、運営者から配布されたアカウント（または登録メールアドレス）でログインし、中間評価（AI Coaching）を受けたり、最終提出を行ったりします。

### 4-1. チームとしてログイン
1. ログイン画面（`http://localhost:5173/login`）へアクセスします。
2. 通常のパスコードログイン、または OIDC（SSO）ログインにてログインします。

### 4-2. プロフィールおよびパスワードの管理
- **「Team Profile」** フォームから、プロダクト名（Product Name）、チーム名（Team Name）、一言キャッチコピー（One-liner Pitch）を設定・更新できます。
- **🔐 パスコード変更**: パスコード認証環境の場合のみ、自身でログインパスコードを変更できます（※OIDC有効時は非表示になります）。

### 4-3. 成果物のアップロードとAI相談（AI Consultation）

本提出前に、AI審査員から中間フィードバック（AI Consultation）を受けることができます。残り相談回数はダッシュボード上で確認できます。

#### 📁 対応している提出形式
提出物は以下の形式に対応しており、**任意の組み合わせで提出可能** です。

1. **ソースコード等の圧縮ファイル (ZIP):**
   - `node_modules` や `.git`、`venv` などの巨大な不要フォルダは必ず除外してZIP化してください。
   - Mac/Linuxでの除外ZIP作成コマンド例: `zip -r submission.zip . -x "node_modules/*" -x ".git/*" -x "venv/*" -x ".next/*"`
2. **プレゼンテーション資料 (PDF)**
3. **デモ動画 (MP4 / MOV):**
   - > [!WARNING]
     > プロジェクトに設定されているLLMプロバイダが OpenAI や Anthropic などの動画非対応プロバイダである場合、動画ファイルをアップロードしようとすると自動的に警告が表示され、提出（送信）ボタンがロックされます。その場合は動画ファイルを除外して提出してください。

#### 提出と相談の手順
1. **ファイルのアップロード:** 画面の「Submit Your Work」エリアにファイルをドラッグ＆ドロップまたは選択して配置します。
2. **AIコーチングを受ける (Submit for AI Consultation):** ボタンをクリックすると、数分でフィードバックが生成されます。

### 4-4. フィードバックとスコアの確認
解析が完了すると、総合スコア、評価バランス（レーダーチャート）、履歴推移（折れ線グラフ）、および有効なAI審査員からのアバター付き個別講評（言語別タブ切り替え可能）が表示されます。

### 4-5. 異議申し立て・質問機能（"Objection!"）
AIの評価結果に納得がいかない、または深く質問したい場合は、画面下部の **「🙋 Objection! / Q&A」** セクションから質問や反論を送信できます。AI審査員同士がディベートを行い、合同回答が生成されます。

### 4-6. 最終成果物としての提出 (Final Submission)
すべての課題が完了したら、成果物ファイルをアップロードした状態で **「Final Submission」** にチェックを入れて最終提出を実行します。最終提出後は成果物の再アップロードやAI相談は行えなくなり、この時点のスコアが主催者のスコアボードに最終スコアとして固定されます。

## 5. 👁️ Observer (オブザーバー / 観客) ガイド

オブザーバー（観客）は、プロジェクト内の各チームの進行状況やスコアボードを安全に監視・閲覧します。

### 5-1. オブザーバーとしてログイン
1. 通常のログイン画面、または OIDC（SSO）ログインにてログインします。

### 5-2. 閲覧可能エリアと機能制限（閲覧専用モード）
画面上部には **「Observer Mode: Read-Only View」** と表示され、一切の書き込み・変更操作が制限されます。

- **閲覧可能な情報**:
  - **チームダッシュボードの閲覧**: 左側のサイドバーから任意のチームを選択し、提出履歴、AIフィードバック、ディベート履歴（Objection!）などを詳細に閲覧できます。
  - **リーダーボード（The Hype Board）の閲覧**:
    - **🚀 Current Rankings**: チームのリアルタイム総合スコアランキング。
    - **🏅 Category Leaders**: 評価基準（Criteria）ごとのタブ切り替えによる部門別ランキング（上位5チーム）。
    - **🤖 Meet the AI Jury Panel**: 有効なAI審査員カードの一覧（「🧠 View Persona」をクリックしてアコーディオン展開することで、各ペルソナの詳細システムプロンプトの確認が可能）。
    - **⚖️ The Rules of the Game**: 設定されているすべての評価基準、ウェイト、詳細説明の一覧。

## 6. ❓ FAQ & トラブルシューティング

#### Q. OIDC (SSO) ログインを試みた際、「Account Not Registered」のエラーになります。
- **A.** あなたのメールアドレスが、Super Admin または Project Admin によって事前にデータベース（チーム、または管理者として）に登録されていない可能性があります。主催者にメールアドレスが正しく登録されているか確認してください。

#### Q. 動画ファイルがアップロードできず、提出ボタンが押せません。
- **A.** プロジェクトのバックエンドが動画解析に非対応のLLM（OpenAI, Anthropic など）を使用するように設定されている可能性があります。動画ファイルを除外して、ZIPやPDFのみで提出を行ってください。

#### Q. ZIPファイルのアップロードで容量エラーになります。
- **A.** 合計ファイルサイズが200MBを超えていないか確認してください。`node_modules` や `.git`、`venv` などの巨大な管理用フォルダが含まれたままになっている場合は、除外コマンドなどを用いて軽量なZIPファイルを再作成してください。
